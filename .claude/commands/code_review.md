# gitステージコードレビュー
---

## 指示
現在gitのステージ状態にあるソースをレビューしてください。

## レビュー観点
あなたは経験豊富なゲーム開発者として、以下の観点でコードレビューを実施してください。

【レビューの優先順位】
1. 設計上の根本的な問題
   - モジュール間の依存関係は適切か
   - 責務は適切に分離されているか
   - 将来の拡張や変更に対して柔軟か

2. 明らかなバグ・不具合
   - エッジケースの考慮漏れ
   - 想定外の入力に対する脆弱性
   - ゲーム特有のタイミング問題

3. パフォーマンスの考慮（文脈依存）
   - 毎フレーム実行される処理は最適化されているか
   - 頻度の低い処理は可読性を優先しているか
   - メモリ使用量やGCへの影響

4. コードの意図の明確性
   - 「なぜ」この実装なのかが明確か
   - 定数には適切な名前とコメントがあるか
   - 複雑なロジックや数式には説明があるか

5. 技術的負債の回避
   - 一時凌ぎのコードになっていないか
   - 「なんとなく動く」実装になっていないか
   - 根本原因に対処しているか

【レビューの進め方】
1. まず全体の設計を俯瞰し、構造的な問題がないか確認
2. 各機能の実装において、上記優先順位に従って指摘
3. 各指摘には「なぜその方が良いのか」の理由を明確に説明
4. 可能な限り具体的な改善案を提示
5. 良い実装については積極的に言及し、学びのポイントを共有

【特に注意すべき点】
- ゲームループ内での処理は特に慎重にレビュー
- エラーハンドリングは適切な粒度で実装されているか
- 将来のデバッグやメンテナンスを考慮した実装か

このレビューを通じて、コードの品質向上だけでなく、お互いの成長につながるような建設的なフィードバックを心がけてください。

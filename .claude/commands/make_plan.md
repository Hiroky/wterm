# 計画書作成プロンプト

## 目的
このプロンプトは、ソフトウェア開発プロジェクトの詳細な計画書を作成するためのものです。作成された計画書は、以下の要件を満たす必要があります：

1. **詳細度**: 誰が見ても同じ実装となるような粒度で記述
2. **タスク管理**: 進捗を細かく管理できるようにタスクを分割
3. **可読性**: 疑似コードを盛り込んでわかりやすく説明
4. **AI作業支援**: AIが効率的に作業に取り組める構造

## 計画書の構造

### 1. プロジェクト概要
- **プロジェクト名**: 明確なプロジェクト名
- **目的**: 何を実現するのか、具体的な目標
- **スコープ**: 含まれる機能、含まれない機能
- **前提条件**: 必要な環境、依存関係、制約事項

### 2. 要件定義
- **機能要件**: 実装する機能の詳細リスト
- **非機能要件**: パフォーマンス、セキュリティ、可用性などの要件
- **インターフェース**: API、UI、データ構造などの仕様
- **データモデル**: 使用するデータ構造、スキーマ

### 3. アーキテクチャ設計
- **全体アーキテクチャ**: システムの構造図（テキストベース）
- **コンポーネント分割**: 各モジュールの責任分担
- **データフロー**: データの流れを時系列で記述
- **依存関係**: コンポーネント間の依存関係

### 4. 実装計画
- **フェーズ分け**: 開発を論理的なフェーズに分割
- **タスク分解**: 各フェーズを具体的なタスクに分解
- **優先順位**: タスクの実行順序と依存関係
- **見積もり**: 各タスクの工数見積もり

### 5. 詳細設計
各コンポーネントについて以下を記述：

#### 5.1 クラス/モジュール設計
- **クラス図**: テキストベースのクラス図
- **メソッド仕様**: 各メソッドの詳細仕様
  - 入力パラメータ
  - 戻り値
  - 処理内容（疑似コード）
  - 例外処理

#### 5.2 アルゴリズム設計
- **処理フロー**: フローチャート（テキストベース）
- **疑似コード**: 実際のコードに近い詳細な疑似コード
- **状態遷移**: 状態変化の詳細

#### 5.3 データ構造設計
- **データモデル**: 構造体の定義
- **データフロー**: データの変換処理

### 6. テスト計画
- **テスト戦略**: 単体テスト、統合テスト、システムテスト
- **テストケース**: 具体的なテストケース一覧
- **テストデータ**: テストで使用するデータ
- **テスト自動化**: CI/CDでのテスト実行計画

### 7. リスク管理
- **リスク一覧**: 潜在的なリスクと影響度
- **対策**: 各リスクに対する対策
- **バックアッププラン**: 問題発生時の対応

### 8. 進捗管理
- **マイルストーン**: 重要なチェックポイント
- **KPI**: 進捗測定指標
- **レビュー**: コードレビュー、設計レビューのタイミング
- **ドキュメント**: 作成するドキュメント一覧

## 記述ガイドライン

### 詳細度の基準
- **コードレベル**: クラス名、メソッド名、変数名まで具体的に
- **処理レベル**: if文、for文の条件まで記述
- **データレベル**: 構造体フィールド、配列サイズまで定義

### タスク分割の基準
- **1タスク = 1-2時間**: 細かく分割して進捗管理しやすく
- **独立性**: 他のタスクに依存しない単位
- **検証可能性**: 完了条件が明確に定義可能

### 疑似コードの書き方
```csharp
public static Result ProcessData(Data input)
{
    // 入力データの検証
    if (!input.IsValid()) {
        throw new ValidationException("Invalid input data");
    }

    // データ処理
    var processed = TransformData(input);

    // 結果の検証
    if (!processed.MeetsCriteria()) {
        return new ErrorResult("Processing failed");
    }

    return new SuccessResult(processed);
}
```

### AI作業支援のための配慮
- **具体性**: 抽象的な表現を避け、具体的な値や名前を使用
- **構造化**: 情報を階層的に整理
- **参照性**: 他のセクションへのクロス参照を明記
- **検証性**: 各記述が検証可能であること

## 品質チェックリスト

計画書作成後に以下の項目を確認：

- [ ] すべての機能がコードレベルで記述されている
- [ ] タスクが1-2時間の単位で分割されている
- [ ] 疑似コードが実際のコードに近い詳細度
- [ ] 依存関係が明確に記述されている
- [ ] テストケースが具体的に記述されている
- [ ] リスクと対策が網羅されている

## 使用方法

1. このプロンプトをAIに提供
2. プロジェクトの具体的な要件を追加
3. AIに計画書の作成を依頼
4. 作成された計画書をレビュー・修正
5. 計画書に基づいて開発を開始
